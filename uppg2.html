Svar i ordning
  1. Element är en funktion inom html dokumentet. De kan antingen vara normala element, eller tomma element. Ett normalt element har en sluttag, men tomma har det inte. Som exempel länktaggen A avslutas med ett /A, och är därför en normal element. Men om vi lägger in en bild, anvander vi oss bara av IMG. Vi behöver ingen sluttag, därför är denna en tom tag.
  2. HTTP är just ett kommunikationsprotokoll. Dvs ett standarsätt att överföra data. HTTP står för Hypertext transfer protocol, och används för att överföra hemsidor från server till din webbläsare. Ett annat exempel är FTP, som förr var vanligt när man skickade filer.
  3. W3C, World Wide Web Consortium är den organisation som beslutar om vilka standarder som ska användas på internet samt utveckla dessa.
  4. WWW eller World wide web är det vi använder vid vardagssurf. Genväg, om det står WWW i URLen så är det webben. HTTP är protokollet som oftast används för att överföra data.
  5. URL. Och det står för Uniform Resource Locator. HTTP:// är det protokoll som används, WWW.facebook.com är domänen och /home.php är resursen som ska läsas.
  6. FTP eller file transfer protocol är ett äldre sätt att överföra större filer. T.ex. program eller media
  7. HTTP bygger på förfrågan och svar förfarande. En klient initierar en förfrågan och en server svarar.
  8. HTML eller Hypertext markup language är det ”språk” som används for att presentera information på webben. Det är ett standardiserat kodspråk.
  9. Det gör det lättare att få överblick över kod samt felsöka och förstå vad som händer.
  10. Indentering påverkar inte utseendet på hemsidan när den läses av en webbläsare.
  11. Kod är nyttigt att förstå för att man ska kunna felsöka på ett behjälpligt sätts, samt att kunna reducera onödig kod.
  12. HTML kommenterat kan man använda för att hänvisa till vart en kod kommer ifrån, man kan kommentera om funktionen som en kod gör, man kan även utan att ta bort kod dölja den för en webbläsare tillfälligt i testnings-syfte</<ol>
